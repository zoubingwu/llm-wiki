# Open Source Trust Erosion

开源信任侵蚀（Open Source Trust Erosion）指一个开源或开放生态产品在保留“开放”外观的同时，通过价格、承诺文本、治理信号、API 和客户端策略的渐进变化，逐步改变用户与产品之间的信任关系。

在 [[The Quiet Renovation at Bitwarden - ByteHaven - Where I ramble about bytes]] 中，[[Bitwarden]] 的案例由几组小变化构成：Premium 涨价、CEO 更替、“Always free” 文案变化、GRIT 价值观改写，以及对 [[Vaultwarden]] 兼容性的长期疑问。

## 识别信号

- 价格变化被放进功能公告，而价格本身缺少独立解释。
- 领导层变化通过 LinkedIn 等间接渠道被发现，公司公告滞后。
- 承诺性文案从产品页面消失，价值观文本发生替换。
- 旧内容被半更新，正文和解释段落出现冲突。
- 第三方兼容项目的长期稳定性依赖上游 API 和客户端策略。

## 为什么重要

开源产品的用户信任通常来自源码可见、社区参与、承诺稳定和退出路径。信任侵蚀会改变用户的风险模型：用户开始评估自托管、数据迁移、分叉、商标风险和客户端可替代性。

这个概念连接 [[Open Source Software]]、[[Self-hosted Password Management]]、[[API Compatibility Drift]] 和 [[Open Source Fork Safety]]。
