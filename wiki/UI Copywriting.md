# UI Copywriting

界面文案（UI Copywriting）用短文本帮助用户理解动作、状态、错误和后果。它覆盖 microcopy、CTA、错误信息、成功提示、帮助文本、语气和数字格式。

## 行动与状态

微文案（Microcopy）包括 label、error message、placeholder、tooltip 等短 UI 文本。CTA（Call to Action）应直接命名动作，例如 Save changes 比 Submit 更具体。成功信息（Success Message）应短而明确，例如 Saved。

## 错误与帮助

错误信息（Error Message）应说明发生了什么以及如何修复。内联错误（Inline Error）把错误放在对应字段旁，减少用户寻找问题的成本。上下文帮助（Contextual Help）把解释放在用户需要的位置，通常一句话就能降低疑惑。

## 表单文本

占位符（Placeholder）适合作为输入提示；持久 label 承担字段身份。数字格式（Numeric Formatting）需要匹配用户 locale，并在 1,000、3.5k、$1.00 等格式之间保持一致。

## 语气与扫描

Voice 是产品文字的一致人格，tone 是同一人格在不同场景里的语气变化。扫描性（Scannability）要求短句、主动语态和信息前置。Front-loading 把最重要词放在开头，让用户快速定位。

## 高风险动作

破坏性语言（Destructive Language）应清晰命名动作，例如 delete、remove、disconnect、revoke。不可逆动作需要让用户准确理解后果。
